# 敏感词告警管理员页面

## 在 VS Code 中运行

1. 使用 VS Code 打开项目根目录 `项目三企智通`。
2. 打开 VS Code 终端并进入前端目录：

   ```powershell
   cd frontend
   ```

3. 启动前端：

   ```powershell
   npm.cmd run dev
   ```

4. 在浏览器访问：

   ```text
   http://localhost:5173/admin/alerts
   ```

如果本机还没有安装依赖，先在 `frontend` 目录执行 `npm.cmd install`，再启动项目。

## 页面包含的功能

- 管理后台左侧新增“告警”入口，并显示待处理数量。
- 首页统计今日告警、待处理、今日已通知和启用规则数量。
- 支持按员工姓名/工号、部门、触发日期范围、处理状态组合筛选。
- 告警详情展示员工、部门、达到阈值的具体时间、敏感词及上下文、通知和处理历史。
- 待处理告警可一键标记为已处理，对应员工的前端演示累计次数立即清零。
- 页面进入后模拟实时弹出“您有一条新的敏感词通知”，顶部铃铛可查看待处理通知。
- 内置“一天内触发 3 次”的系统默认规则，管理员也可修改、启停或删除该默认规则。
- 管理员可新增和编辑规则，周期支持一天、一周或自定义正整数天数，次数支持任意正整数。
- 规则固定通过站内通知和管理员邮件两个渠道发送。

当前告警数据与规则数据是前端演示数据，集中放在 `src/views/AlertCenterView.vue`。后续后端提供告警接口后，可将该文件内的初始化数据和操作方法替换为接口请求，页面结构与交互无需重做。
